## Overview

Welcome to the official plugin registry for the Final Fantasy VI Pixel Remastered Save Editor. This repository hosts community-contributed plugins that extend the functionality of the save editor.

### What are Plugins?

Plugins are Lua scripts that can:
- Read and analyze save data
- Modify character stats, inventory, and party composition
- Automate repetitive tasks
- Provide advanced analysis and optimization tools
- Extend the editor with custom features

### Plugin System Features

- **Sandboxed Execution:** Plugins run in a secure Lua VM with restricted access
- **Permission-Based Security:** Plugins declare required permissions (read/write/UI)
- **Comprehensive API:** Full access to characters, inventory, party, equipment, and more
- **Easy Installation:** One-click installation through the editor's marketplace
- **Automatic Updates:** Optional auto-update support for plugins
- **Community Driven:** Anyone can contribute plugins

## Getting Started

### For Users

#### Installing Plugins

1. **Open the Editor:** Launch the FF6 Save Editor
2. **Open Marketplace:** Go to `Tools → Marketplace` or `Community → Marketplace`
3. **Browse Plugins:** Search or filter plugins by category
4. **Install:** Click "Install" on any plugin
5. **Use:** Access installed plugins via `Tools → Plugin Manager`

#### Managing Plugins

- **Enable/Disable:** Toggle plugins on/off without uninstalling
- **Configure:** Some plugins have configurable settings
- **Update:** Check for updates via `Tools → Check for Plugin Updates`
- **Remove:** Uninstall plugins through the Plugin Manager

### For Developers

#### Quick Start: Creating Your First Plugin

1. **Review Documentation:**
   - [PLUGIN_SPECIFICATION.md](PLUGIN_SPECIFICATION.md) - Plugin format specification
   - [Plugin API Reference](https://github.com/ff6-save-editor/editor/blob/main/PHASE_4B_API_REFERENCE.md)
   - [Plugin Development Guide](https://github.com/ff6-save-editor/editor/blob/main/PHASE_4B_PLUGIN_GUIDE.md)

2. **Use a Template:**
   ```lua
   -- Plugin Metadata (required comments)
   -- @id: my-first-plugin
   -- @name: My First Plugin
   -- @version: 1.0.0
   -- @author: Your Name
   -- @description: A simple example plugin
   -- @permissions: read_save, ui_display
   
   function main(api)
       -- Get first character
       local char = api:GetCharacter(0)
       if char then
           local msg = string.format(
               "Character: %s\nLevel: %d\nHP: %d/%d",
               char.Name, char.Level, char.CurrentHP, char.MaxHP
           )
           api:ShowDialog("Character Info", msg)
       end
       return true
   end
   ```

3. **Test Locally:**
   - Place plugin in `~/.ff6editor/plugins/`
   - Open Plugin Manager in editor
   - Enable and run your plugin

4. **Submit to Registry:**
   - Fork this repository
   - Add your plugin to `plugins/your-plugin-name/`
   - Create pull request (see [CONTRIBUTING.md](CONTRIBUTING.md))

## Plugin Permissions

Plugins must declare the permissions they require:

| Permission | Description | Risk Level |
|------------|-------------|------------|
| `read_save` | Read save data (characters, inventory, etc.) | Low |
| `write_save` | Modify save data | Medium |
| `ui_display` | Show dialogs and prompts to user | Low |
| `events` | Register event hooks | Low |

**Note:** Plugins cannot access the filesystem, network, or operating system. They run in a sandboxed Lua environment.

## Security

### Sandboxing

All plugins run in a secure Lua VM with:
- **Restricted Standard Library:** Only safe modules (table, string, math, utf8)
- **No File I/O:** Plugins cannot read/write files
- **No Network Access:** Plugins cannot make network requests
- **No OS Commands:** Plugins cannot execute system commands
- **Execution Timeout:** 30 seconds maximum execution time
- **Memory Limit:** 50MB maximum memory usage

### Review Process

All submitted plugins undergo:
1. **Automated Validation:** Syntax, metadata, checksum verification
2. **Security Review:** Check for forbidden API calls or patterns
3. **Code Quality Review:** Best practices, error handling, documentation
4. **Testing:** Verify plugin works as described
5. **Approval:** Maintainer approval required before publication
